# План спринта (Lab 2)

## Цель спринта
Подготовить и продемонстрировать рабочий прототип веб-системы предварительной оценки риска диабета 2 типа на основе требований Lab 1.

## Задачи спринта
| № | Задача | Приоритет | Ожидаемый результат |
|---|---|---|---|
| 1 | Уточнить границы MVP и зафиксировать архитектуру | Высокий | Подготовлен документ `architecture.md` |
| 2 | Подготовить дизайн-заметки по решениям Lab 2 | Высокий | Подготовлен документ `design-notes.md` |
| 3 | Описать baseline-логику оценки риска | Высокий | Подготовлен документ `ai-deliverables.md` |
| 4 | Реализовать backend endpoint `GET /health` | Высокий | Проверка доступности API |
| 5 | Реализовать backend endpoint `POST /risk-assessment` | Высокий | JSON-ответ с score/risk_level/recommendation |
| 6 | Реализовать модуль rule-based расчета риска | Высокий | Прозрачные правила начисления баллов |
| 7 | Подготовить простую frontend-страницу анкеты | Средний | Форма ввода и отправка запроса в API |
| 8 | Вывести результат и дисклеймер в интерфейсе | Средний | Пользователь видит итог оценки и рекомендации |
| 9 | Описать локальный запуск и структуру проекта | Средний | Обновлен `README.md` |

## Ожидаемые результаты спринта
- Сформирован единый комплект документов Lab 2.
- Реализован минимальный рабочий прототип (frontend + backend).
- Подготовлена база для расширения функциональности в Lab 3.

## Риски и меры снижения
| Риск | Влияние | Митигирующее действие |
|---|---|---|
| Несогласованность документов и кода | Среднее | Сверка терминов с Lab 1 перед сдачей |
| Ошибки в расчетных правилах | Среднее | Проверка на 3-5 ручных тестовых примерах |
| Нехватка времени на полировку UI | Низкое | Минимальный чистый интерфейс без усложнений |
| Проблемы локального запуска на разных ПК | Среднее | Короткие и явные инструкции в README |
