# Ключевые use case (Lab 1)

## Связь с MVP Lab 3
- В MVP реализованы: **UC-01, UC-02, UC-03, UC-04, UC-05**.
- Сценарий **UC-06** отмечен как возможное расширение и в MVP не входит.

## UC-01. Заполнение анкеты риска
- **Актор:** Пользователь
- **Предусловия:** Пользователь открыл веб-страницу анкеты.
- **Основной сценарий:**
  1. Пользователь вводит данные в поля анкеты.
  2. Система проверяет корректность ввода (обязательные поля, диапазоны значений).
  3. Пользователь нажимает кнопку расчета.
- **Альтернативный сценарий:**
  1. Пользователь оставил обязательное поле пустым или ввел некорректное значение.
  2. Система показывает сообщение об ошибке и просит исправить данные.
- **Результат:** Валидные данные переданы в модуль расчета риска.

## UC-02. Расчет уровня риска
- **Актор:** Система
- **Предусловия:** Получены корректные данные анкеты.
- **Основной сценарий:**
  1. Система присваивает баллы каждому фактору риска.
  2. Система суммирует баллы.
  3. Система определяет категорию риска по пороговым значениям.
- **Альтернативный сценарий:**
  1. Внутренняя ошибка обработки данных.
  2. Система возвращает сообщение о технической ошибке.
- **Результат:** Рассчитаны итоговый балл и категория риска.

## UC-03. Просмотр результата оценки
- **Актор:** Пользователь
- **Предусловия:** Расчет риска успешно выполнен.
- **Основной сценарий:**
  1. Система отображает категорию риска.
  2. Система показывает итоговый балл.
  3. Система выводит краткое пояснение по факторам риска.
- **Альтернативный сценарий:**
  1. Расчет не завершен из-за ошибки запроса.
  2. Система предлагает повторить отправку анкеты.
- **Результат:** Пользователь получает итог предварительной оценки.

## UC-04. Получение рекомендаций
- **Актор:** Пользователь
- **Предусловия:** Пользователю показан результат оценки риска.
- **Основной сценарий:**
  1. Система выводит рекомендации по категории риска.
  2. Пользователь знакомится с рекомендациями.
  3. Пользователь при необходимости планирует дальнейшие действия.
- **Альтернативный сценарий:**
  1. Для категории риска отсутствует детальный шаблон.
  2. Система выводит универсальную рекомендацию обратиться к врачу.
- **Результат:** Пользователь получает понятные дальнейшие шаги профилактического характера.

## UC-05. Повторная оценка после изменения данных
- **Актор:** Пользователь
- **Предусловия:** Пользователь уже получил результат первичной оценки.
- **Основной сценарий:**
  1. Пользователь изменяет значения в анкете.
  2. Пользователь повторно запускает расчет.
  3. Система выдает обновленный результат.
- **Альтернативный сценарий:**
  1. Пользователь вводит некорректные данные при повторной оценке.
  2. Система просит исправить поля и повторить действие.
- **Результат:** Пользователь получает обновленную предварительную оценку риска.

## UC-06. Настройка параметров оценки (вне MVP)
- **Актор:** Администратор проекта
- **Предусловия:** В системе реализован административный режим (на будущих этапах).
- **Основной сценарий:**
  1. Администратор открывает раздел параметров оценки.
  2. Администратор корректирует пороги/веса факторов.
  3. Система применяет обновленные параметры для новых расчетов.
- **Альтернативный сценарий:**
  1. Введены некорректные значения параметров.
  2. Система отклоняет изменения и показывает ошибку валидации.
- **Результат:** Параметры расчета обновлены для будущих оценок.
