# Ключевые use case

## UC-01. Заполнение анкеты риска
- **Актор:** Пользователь
- **Предусловия:** Пользователь открыл веб-страницу анкеты.
- **Основной сценарий:**
  1. Пользователь вводит данные в поля анкеты.
  2. Система проверяет корректность ввода (обязательные поля, диапазоны значений).
  3. Пользователь нажимает кнопку расчета.
- **Альтернативный сценарий:**
  1. Пользователь оставил обязательное поле пустым или ввел некорректное значение.
  2. Система показывает сообщение об ошибке и просит исправить данные.
- **Результат:** Валидные данные переданы в модуль расчета риска.

## UC-02. Расчет уровня риска
- **Актор:** Система
- **Предусловия:** Получены корректные данные анкеты.
- **Основной сценарий:**
  1. Система присваивает баллы каждому фактору риска.
  2. Система суммирует баллы.
  3. Система определяет категорию риска по пороговым значениям.
- **Альтернативный сценарий:**
  1. Не загружены параметры расчета (ошибка конфигурации).
  2. Система возвращает сообщение о технической ошибке.
- **Результат:** Рассчитаны итоговый балл и категория риска.

## UC-03. Просмотр результата оценки
- **Актор:** Пользователь
- **Предусловия:** Расчет риска успешно выполнен.
- **Основной сценарий:**
  1. Система отображает категорию риска (низкий/средний/высокий).
  2. Система показывает итоговый балл.
  3. Система кратко перечисляет факторы, повлиявшие на результат.
- **Альтернативный сценарий:**
  1. Пользователь прервал процесс до завершения расчета.
  2. Система предлагает вернуться к анкете.
- **Результат:** Пользователь получает понятный итог предварительной оценки.

## UC-04. Получение рекомендаций
- **Актор:** Пользователь
- **Предусловия:** Пользователю показан результат оценки риска.
- **Основной сценарий:**
  1. Система выводит базовые рекомендации в зависимости от категории риска.
  2. Пользователь знакомится с рекомендациями.
  3. Пользователь при необходимости переходит к повторному заполнению анкеты.
- **Альтернативный сценарий:**
  1. Для выбранной категории отсутствует шаблон текста.
  2. Система показывает универсальную рекомендацию обратиться к врачу.
- **Результат:** Пользователь получает дальнейшие шаги профилактического характера.

## UC-05. Сохранение результата в истории
- **Актор:** Пользователь
- **Предусловия:** Пользователь авторизован; результат оценки получен.
- **Основной сценарий:**
  1. Пользователь нажимает «Сохранить результат».
  2. Система сохраняет дату, итоговый балл и категорию риска.
  3. Пользователь открывает список прошлых оценок.
- **Альтернативный сценарий:**
  1. Пользователь не авторизован.
  2. Система предлагает войти в аккаунт или продолжить без сохранения.
- **Результат:** Результат сохранен и доступен для последующего просмотра.

## UC-06. Настройка параметров анкеты
- **Актор:** Администратор
- **Предусловия:** Администратор вошел в систему с правами управления.
- **Основной сценарий:**
  1. Администратор открывает раздел настроек.
  2. Администратор изменяет веса факторов или пороги категорий риска.
  3. Система сохраняет обновленные параметры.
- **Альтернативный сценарий:**
  1. Введены некорректные значения параметров.
  2. Система отклоняет изменения и показывает ошибку валидации.
- **Результат:** Параметры расчета обновлены для последующих оценок.
