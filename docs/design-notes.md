# Дизайн-заметки (Lab 2)

## Контекст
В Lab 1 были зафиксированы требования к учебной системе предварительной оценки риска диабета 2 типа.
В Lab 2 сделан переход к проектным решениям и прототипу, сохраняя ограниченный и реалистичный объем MVP.

## Основные проектные решения
1. Выбрана двухуровневая архитектура: frontend + backend API.
2. Расчет риска реализован в виде прозрачной rule-based логики.
3. Данные обрабатываются без базы данных, только в рамках одного запроса.
4. Интерфейс ограничен одной основной страницей с анкетой и блоком результата.

## Почему выбрана легковесная архитектура
- Проект является учебным, а не промышленным.
- Основная цель текущего этапа: продемонстрировать полный пользовательский сценарий.
- Простая структура снижает вероятность ошибок и ускоряет подготовку к защите лабораторной.

## Допущения для MVP
- Пользователь вводит корректные базовые данные о возрасте, росте, весе и привычках.
- Оценка риска носит ориентировочный характер.
- Рекомендации формируются по шаблону, без персонализированной медицинской интерпретации.

## Ключевые элементы UI
- Заголовок и краткое описание назначения системы.
- Форма анкеты с базовыми параметрами риска.
- Кнопка расчета.
- Блок результата с тремя полями: балл, уровень риска, рекомендация.
- Явный дисклеймер о недиагностическом характере результата.

## Границы backend-модулей
- `main.py` — маршруты API и конфигурация приложения.
- `schemas.py` — модели входных/выходных данных и базовая валидация.
- `risk_logic.py` — правила начисления баллов и определение категории риска.

## Принятые упрощения в Lab 2
- Нет авторизации и личного кабинета.
- Нет истории предыдущих оценок.
- Нет подключения к внешним медицинским системам.
- Нет обучения модели на реальных датасетах.

## Ограничения текущего подхода
- Rule-based оценка не учитывает все клинические факторы.
- Результат нельзя использовать как медицинское заключение.
- При изменении анкеты требуется ручная корректировка правил.

## Что переносится в Lab 3
- Расширение набора тестов (API и UI).
- Улучшение структуры рекомендаций для разных профилей пользователей.
- Обсуждение варианта использования валидационных датасетов для сравнения с baseline-подходом.
