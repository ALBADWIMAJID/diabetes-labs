# Разработка системы оценки риска диабета 2 типа

## О проекте

Данный репозиторий содержит последовательные лабораторные работы по учебному проекту веб-системы **предварительной оценки риска диабета 2 типа**.

Проект выполнен в рамках дисциплины **«Системы искусственного интеллекта»** и демонстрирует полный путь разработки: от анализа предметной области и формализации требований до проектирования архитектуры и реализации MVP-прототипа.

> **Важно:** система не является медицинским диагностическим инструментом, не заменяет консультацию врача и предназначена исключительно для учебных целей.

---

## Цель проекта

Разработать простую и наглядную веб-систему, которая позволяет пользователю пройти анкетирование, получить **предварительную оценку уровня риска** и ознакомиться с краткими рекомендациями на основе введенных данных.

---

## Этапы разработки

Проект организован в виде последовательных лабораторных работ:

### **Lab 1 — Анализ**
На данном этапе была выполнена формализация предметной области, определены требования, основные сценарии использования и роли участников проекта.

Результаты:
- описание проблемы и целей системы;
- определение целевой аудитории;
- формирование пользовательских сценариев;
- анализ заинтересованных сторон;
- построение матрицы ответственности.

### **Lab 2 — Проектирование**
На этом этапе была подготовлена архитектурная и проектная основа будущего решения.

Результаты:
- описание архитектуры системы;
- обоснование проектных решений;
- планирование спринта;
- разработка baseline-логики оценки риска;
- подготовка технического каркаса frontend и backend.

### **Lab 3 — MVP-прототип**
На данном этапе реализован минимально жизнеспособный прототип системы с рабочим сквозным пользовательским сценарием.

Результаты:
- frontend-интерфейс анкеты;
- backend API на FastAPI;
- rule-based логика расчета риска;
- интеграция клиентской и серверной частей;
- формирование итогового результата для пользователя.

---

## Основные возможности MVP

В текущей реализации система поддерживает:

- ввод пользовательских параметров через веб-форму;
- отправку данных на backend API;
- расчет итогового балла риска;
- определение категории риска:
  - **низкий**
  - **средний**
  - **высокий**
- вывод краткой рекомендации;
- отображение пояснения к полученному результату.

---

## Основной пользовательский сценарий

1. Пользователь открывает веб-страницу системы.
2. Заполняет анкету (возраст, рост, вес, физическая активность и дополнительные факторы).
3. Нажимает кнопку расчета.
4. Frontend отправляет данные в backend через `POST /risk-assessment`.
5. Backend обрабатывает данные и рассчитывает итоговый балл.
6. Пользователь получает:
   - `score`
   - `risk_level`
   - `recommendation`
   - `explanation`

---

## Архитектура решения

Проект реализован в виде упрощенной двухуровневой архитектуры:

- **Frontend** — интерфейс пользователя для ввода данных и отображения результата;
- **Backend** — API и модуль расчета риска;
- **Risk Logic Module** — rule-based логика оценки риска, вынесенная в отдельный сервисный слой.

### Взаимодействие компонентов

`Пользователь → Frontend → Backend API → Модуль расчета риска → Backend API → Frontend → Результат`

---

## Реализованные API-эндпоинты

### Backend (FastAPI)

- `GET /health` — проверка доступности сервиса;
- `POST /risk-assessment` — обработка анкеты и расчет предварительного риска.
