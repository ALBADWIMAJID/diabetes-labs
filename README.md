# Лабораторная работа №3

## Проект
**Разработка системы оценки риска диабета 2 типа**

## Цель Lab 3
Lab 3 фиксирует MVP-прототип как продолжение Lab 1 (анализ) и Lab 2 (архитектура и подготовка).
На этом этапе реализован и интегрирован полный основной сценарий: от заполнения анкеты до вывода результата оценки риска.

## Что реализовано в MVP
- Минимальный backend на FastAPI:
  - `GET /health`
  - `POST /risk-assessment`
- Rule-based расчет балла риска по анкете (без БД и без авторизации).
- Минимальный frontend с формой анкеты, отправкой запроса и отображением результата:
  - `score`
  - `risk_level`
  - `recommendation`
  - `explanation`
- Документы Lab 3: обновленные требования MVP и отчет по интеграции.

## Основной пользовательский сценарий
1. Пользователь открывает страницу frontend.
2. Заполняет анкету (возраст, рост, вес, активность и дополнительные факторы).
3. Нажимает кнопку расчета.
4. Frontend отправляет данные в backend (`POST /risk-assessment`).
5. Backend вычисляет балл и уровень риска.
6. Пользователь видит результат и краткую рекомендацию.

## Структура репозитория
- `docs/` — документация Lab 1, Lab 2 и Lab 3
- `backend/` — API и логика расчета риска
- `frontend/` — интерфейс анкеты и показ результата
- `.env.example` — пример базовых переменных окружения

## Локальный запуск backend
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

Проверка:
- Health: `http://localhost:8000/health`
- Swagger: `http://localhost:8000/docs`

## Локальный запуск frontend
```bash
cd frontend
python -m http.server 5500
```

Открыть в браузере: `http://localhost:5500`

По умолчанию frontend обращается к backend по адресу `http://localhost:8000`.

## Важное ограничение
Система является учебным инструментом предварительной оценки риска.
Результат не является медицинским диагнозом и не заменяет консультацию врача.
